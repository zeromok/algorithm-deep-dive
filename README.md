# Algorithm Deep Dive

> **"문제를 풀다가 막혔을 때, 그 기법을 파고들기 위한 공간"**

## 💡 왜 이 프로젝트가 존재하는가

알고리즘 문제를 풀다 보면 특정 기법이 부족하다는 걸 깨닫는 순간이 온다. 그럴 때 온라인 저지에서 계속 문제를 풀기보다는, **그 기법 자체를 집중적으로 연습**하는 게 더 효율적이다.

이 프로젝트는 바로 그런 순간을 위한 **기법 집중 연습 환경**이다.

### 핵심 원칙

1. **본질 중심 학습**: 단순히 "동작하는 코드"가 아니라 **"왜 그렇게 동작하는지"** 이해하기
2. **반복 훈련**: 패턴을 몸에 익힐 때까지 의도적으로 반복 구현
3. **실행 흐름 추적**: 디버거를 돌리며 알고리즘의 각 단계가 어떻게 진행되는지 관찰
4. **독립적 실험**: 외부 문제에 구애받지 않고 순수하게 기법만 연습

## 🔄 사용 워크플로우

```
1. 백준/기타 플랫폼에서 문제 풀이
          ↓
2. "이 기법이 약하다" 깨달음 (예: DP, 그래프 탐색)
          ↓
3. algorithm-deep-dive로 전환
          ↓
4. 해당 기법을 집중 연습 (구현 → 실행 → 분석)
          ↓
5. 이해도가 올라간 상태로 문제 풀이 복귀
```

### 실제 활용 예시

**시나리오**: Grid DP 문제를 풀다가 Top-down과 Bottom-up의 차이를 확실히 모르겠다
1. `pattern/dp/` 폴더로 이동
2. `BasicDPV0`, `BasicDPV1`, `BasicDPV2` 구현하며 두 방식 비교
3. 디버거로 메모이제이션과 테이블 빌드업의 차이 관찰
4. 이해한 내용을 바탕으로 원래 문제 해결

## 🗂️ 프로젝트 구조

```
src/main/java/daniel/algorithmdeepdive/
│
├── pattern/              # 알고리즘 패턴 집중 연습
│   ├── dp/              # Dynamic Programming (Top-down vs Bottom-up)
│   ├── bruteforce/      # 완전탐색 (조합, 순열, 백트래킹)
│   ├── greedy/          # 그리디 (최적 선택 전략)
│   ├── sort/            # 정렬 알고리즘 구현
│   └── search/          # 탐색 (이진탐색, 그래프탐색)
│
├── dataStructure/       # 자료구조 직접 구현
│   ├── graph/          # 그래프 표현 (인접행렬 vs 인접리스트)
│   └── heap/           # 힙 구조와 우선순위 큐
│
└── mathematics/         # 알고리즘에 필요한 수학 개념
    ├── directionvector/ # 방향벡터와 2D 그리드 탐색
    ├── GCDAndLCM       # 최대공약수/최소공배수
    └── FloatingPoint   # 부동소수점 연산
```
